# awd-c0iq 最新程序使用教程

## 一、打包方法

### 1. 免 CGO 打包

这是当前推荐的发布方式，产物兼容性更稳：

```powershell
$env:CGO_ENABLED='0'
go build -tags production -o dist\awd-c0iq-nocgo.exe .
```

### 2. 普通打包

```powershell
go build -tags production -o dist\awd-c0iq.exe .
```

### 3. 发布包整理

推荐把以下文件一起放进发布包：

- `awd-c0iq-nocgo.exe`
- `config.example.yaml`
- `使用说明.md`
- `最新程序使用教程.md`
- `RELEASE_NOTES_v1.0.md`

## 二、运行前需要准备什么

### 1. 本机环境

- Windows 环境，直接运行 `exe`
- 如需本地靶场联调，可使用 WSL
- 如果需要自行重新编译，确保本机已安装 Go

### 2. 目标侧最少准备

- 一台可通过 SSH 登录的靶机
- 靶机 IP、SSH 端口、用户名、密码
- 一个可用的 WebShell
- 如果要做流量分析，靶机需要能执行 `tcpdump`
- 如果要做文件管理，SSH 账号需要对目标目录有读写权限

### 3. WebShell 侧最少要明确的参数

- Shell 地址，例如 `/shell.php`
- 请求方式：`GET` 或 `POST`
- 参数名，例如 `pass`、`b`
- 是否带固定查询参数，例如 `a=system`
- 是“执行 PHP 代码”型，还是“直接执行系统命令”型

## 三、程序启动后的正确使用顺序

### 1. 打开程序后先建立 SSH 连接

首次进入会弹出 SSH 登录窗口，需要填写：

- IP 地址
- 端口，默认 `22`
- 用户名，默认 `root`
- 密码

连接成功后：

- 顶部状态栏会显示当前 SSH 会话
- 总览页会进入 SSH 终端和文件管理器
- 防守模块和流量模块会复用这条 SSH 会话

### 2. 到资产模块录入目标和运行配置

资产模块主要负责两件事：

- 维护目标列表
- 保存 WebShell 运行配置和远程工作目录

常见 WebShell 场景配置示例：

#### 场景 A：一句话木马

- 方法：`POST`
- 参数名：`pass`
- 固定查询：留空
- 载荷类型：PHP 代码

#### 场景 B：系统命令型 Shell

请求形式：

```text
http://127.0.0.1:9001/shell.php?a=system
POST: b=whoami
```

应这样配置：

- 方法：`POST`
- 参数名：`b`
- 固定查询：`a=system`
- 载荷类型：原始命令

### 3. 到攻击模块做联通性验证

建议按这个顺序测试：

1. `Shell 测试`
2. `命令执行`
3. `Flag 获取`
4. `载荷攻防`

原因很简单：

- Shell 测试通过，说明基础链路通
- 命令执行有回显，说明参数和方法配对正确
- 然后再跑 Flag 或上传其他载荷，排障会更快

## 四、各模块怎么用

### 1. 总览模块

总览模块现在由两个核心分页组成。

#### SSH 终端会话

用途：

- 执行任意远程命令
- 查看运行环境
- 临时排查站点、权限、进程、网络

建议先执行：

```bash
whoami
pwd
uname -a
ls -la
```

如果终端无输出，优先检查：

- SSH 是否已连接
- 账号是否有 shell 权限
- 目标系统是否限制交互终端

#### 文件管理器

支持：

- 浏览远程目录
- 进入上级目录
- 刷新目录
- 上传文件
- 下载文件
- 删除文件
- 重命名
- 编辑文本文件

推荐优先定位：

- Web 根目录
- 备份目录
- 日志目录
- Flag 可能落点

### 2. 资产模块

资产模块用于维护基础上下文，不直接执行攻击。

主要配置项：

- 目标列表
- WebShell 请求方法
- 参数名
- 固定查询参数
- 远程工作目录

如果你已经通过 SSH 连上靶机：

- 防守模块不再需要单独填写“本机 IP”
- 流量模块也不再需要单独填写“本机 IP”
- 它们默认取当前 SSH 连接的主机

### 3. 攻击模块

攻击模块包含四类常用操作。

#### Shell 测试

作用：

- 批量验证目标是否存活
- 判断 WebShell 参数是否匹配

输出文件：

- `output/success.txt`
- `output/error.txt`

现在这两个文件都是追加写入，不会覆盖旧结果。

#### 命令执行

作用：

- 对当前目标直接下发命令
- 查看标准输出回显

如果显示“执行成功但无回显”，通常说明：

- 命令本身没有输出
- WebShell 是盲执行
- 参数类型不匹配
- 当前 Shell 不支持 PHP 代码执行

#### Flag 获取与提交

作用：

- 读取目标上的 Flag
- 结果写入 `output/flag.txt`

`flag.txt` 现在已改为追加模式：

- 新抓到的 Flag 不会覆盖旧 Flag
- 每条 Flag 单独占一行

#### 载荷攻防

这个分页已经放在攻击模块中，主要用于：

- 编辑 Undead 载荷
- 编辑 MD5 载荷
- 编辑 Worm 载荷
- 直接在界面中修改并保存相关代码

如果当前基础 Shell 是“原始命令型”而不是“PHP 代码型”：

- 普通执行命令可正常使用
- 但依赖 PHP 片段写文件的上传链路可能不可用

### 4. 防守模块

防守模块默认只针对当前 SSH 主机执行。

典型用途：

- 网站备份
- 数据库备份
- 查杀恶意文件
- 恢复站点
- 加固站点

使用前建议先在终端确认：

- 当前连接确实是我方服务器
- SSH 账号有权限访问站点目录
- 远程工作目录配置正确

### 5. 流量模块

流量模块现在走 SSH 远程抓包。

抓包逻辑：

- 通过当前 SSH 会话在目标机上启动 `tcpdump`
- 只抓当前 SSH 主机对应的入站流量
- 本地解析抓包结果并展示到 UI

#### 实时抓流量

支持：

- 启动抓包
- 停止抓包
- 查看实时记录
- 自动识别常见协议

当前会自动识别：

- HTTP
- HTTPS
- SSH
- DNS
- MySQL
- Redis
- TCP
- UDP

不需要手工指定“这是 HTTP”还是“这是 TCP”，程序会尽量自动判断。

#### 抓包记录

支持：

- 历史记录列表
- 关键字搜索
- 刷新
- 右侧详情预览

搜索框支持 `&` 做联合查询，例如：

```text
80&post&172.27.132.236
```

含义是：

- 结果里同时包含 `80`
- 同时包含 `post`
- 同时包含 `172.27.132.236`

#### HTTP 详情预览

对于明文 HTTP，请求详情中可以看到：

- 请求行
- 请求头
- 常见 POST 请求体

说明：

- 这不是完整的 Wireshark 替代品
- 对 HTTPS 无法直接看到明文请求体
- 对复杂 TCP 重组、压缩、分块传输的支持不如专业抓包工具完整

## 五、输出文件和日志怎么看

常用输出位置如下：

- `output/flag.txt`：Flag 结果
- `output/error.txt`：Shell 测试失败或运行错误
- `output/success.txt`：Shell 测试成功结果
- `log/`：程序运行日志
- `pcap/`：抓包文件

当前日志增强点：

- HTTP 请求报错会记录 URL
- 会记录请求方法
- POST 请求会记录请求体
- 会记录响应状态码
- 会记录详细错误原因

这类日志更适合排查：

- 超时
- 404 / 500
- 请求参数写错
- 代理或转发异常

## 六、最常见的排障顺序

遇到“功能点不通”时，按这个顺序查：

1. 先看顶部 SSH 状态是否正常
2. 在总览终端执行 `whoami`、`pwd`、`ls`
3. 在资产模块确认 WebShell 方法、参数名、固定查询是否正确
4. 在攻击模块先跑 Shell 测试
5. 再执行一条简单命令，例如 `id` 或 `whoami`
6. 如果是流量模块没数据，先在终端检查 `tcpdump` 是否存在
7. 最后再看 `output/error.txt` 和 `log/` 里的详细报错

## 七、发布和上传建议

如果要把当前版本同步到 GitHub，建议：

1. 源代码正常推送到 `main`
2. 发布包单独放到 `package/` 目录，或上传到 GitHub Release
3. 对应版本打 tag，例如 `v1.0`
4. 同步上传本教程和版本说明

当前 `v1.0` 更推荐上传的发布文件是：

- `package/awd-c0iq-v1.0-windows-amd64.zip`

## 八、注意事项

- 只在已授权的 AWD / CTF / 靶场环境中使用
- WebShell 参数、方法、固定查询三者必须匹配
- 流量模块依赖远端 `tcpdump`
- HTTPS 不能直接看到明文请求体
- 发布给队友时，优先发 zip 包，不建议直接发散装运行目录
